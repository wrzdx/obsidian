## 1. Callbacks

### Что такое callback

Callback — это функция, переданная в другую функцию как аргумент, которая затем вызывается внутри внешней функции для завершения какого-то действия. 
Пример:

```js
myDiv.addEventListener("click", function(){
  // что-то делаем!
})
```

Здесь функция-обработчик передаётся в `addEventListener()`, и будет вызвана при клике. 

### Проблемы с callback’ами

Когда требуется выполнить серию асинхронных действий в определённом порядке — например: сначала A, затем B, затем C — часто возникает «callback hell» (ад колбэков) — глубоко вложенные функции, плохая читаемость, сложности с обработкой ошибок.

### Использование

Подходит для простых случаев: когда нужно просто «после этого вызвать вот эту функцию». Но если логика становится сложной — лучше другие подходы.

## 2. Promises

### Что такое Promise

Promise — объект, который может быть результатом (или ошибкой) в будущем. 
Простой пример:

```js
const getData = function() {
  // делаем fetch…
  // возвращаем данные
  return data
}
const myData = getData();
const pieceOfData = myData['whatever'] // → проблема: данные ещё не получены
```

Чтобы исправить это: если `getData()` возвращает Promise, можно сделать:

```js
const myData = getData(); // Promise
myData.then(function(data){
  const pieceOfData = data['whatever'];
});
```


### Какие состояния у Promise

- **pending** (ожидание)
- **fulfilled** (успешно выполнен)
- **rejected** (ошибка)  

### Метод `.then()`

Метод `.then()` даёт возможность указать: когда Promise выполнится успешно — что делать; либо (в сочетании с `.catch()`) — при ошибке. Пример:

```js
promise.then(function(value){
  // успешно: value
}).catch(function(error){
  // ошибка: error
});
```

Также `.then()` помогает цепочкам (chaining) асинхронных операций.

### Почему Promises лучше в некоторых случаях

- Улучшают читаемость по сравнению с вложенными callback’ами
- Упрощают обработку ошибок
- Легче комбинировать/цеплять асинхронные операции

## 3. Event Loop

JavaScript работает в однопоточном режиме (single-threaded), но окружение (браузер или Node.js) предоставляет API для асинхронных задач (например, таймеры, запросы, события). Event Loop (“штука”, которая следит за тем, когда можно вернуться к очереди задач) обеспечивает, что асинхронные операции не блокируют главный поток.

**Ключевые понятия:**

- Call Stack (стек вызовов)
- Task Queue / Callback Queue (очередь задач)
- Microtask Queue (например, задачи Promise)
