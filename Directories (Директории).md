Директория — это тоже файл, но специальный. В нем хранится список других файлов.

#### 4.2.1 Single-Level Directory Systems (Одноуровневые)
*   **Концепция:** Одна корневая папка ("свалка") для всех файлов.
*   **История:** Первые ПК (CP/M), суперкомпьютер CDC 6600.
*   **Где сейчас:** Простые устройства (старые цифровые камеры, mp3-плееры).
*   **Проблема:** Если файлов много (тысячи), найти что-то невозможно. Нельзя создать два файла с именем `data.txt`.

#### 4.2.2 Hierarchical Directory Systems (Иерархические)
*   **Концепция:** Дерево. Есть корень (`/`), в нем папки, в папках подпапки.
*   **Польза:** Группировка файлов по смыслу (проекты, пользователи).
*   **История:** Придумано в MULTICS, используется везде (Windows, UNIX).

#### 4.2.3 Path Names (Пути)
Как обратиться к файлу в дереве? Два способа:
1.  **Absolute Path (Абсолютный путь):**
    *   Полный адрес от корня.
    *   Windows: `C:\Windows\System32\drivers`
    *   UNIX: `/usr/bin/gcc`
    *   Всегда уникален.
2.  **Relative Path (Относительный путь):**
    *   Адрес относительно **Current Working Directory (Текущей рабочей директории)**.
    *   Если ты в `/usr/`, то путь `bin/gcc` приведет к `/usr/bin/gcc`.
    *   **`.` (точка):** Текущая директория.
    *   **`..` (две точки):** Родительская директория.

*Пример:* Команда `cp ../lib/dict .` копирует файл из папки выше (`../lib`) в текущую (`.`).

#### 4.2.4 Directory Operations (Операции с директориями)
1.  **Create (mkdir):** Создать новую папку. В ней сразу появляются `.` и `..`.
2.  **Delete (rmdir):** Удалить. Обычно разрешено удалять только **пустые** папки.
3.  **Opendir / Closedir:** Открыть для чтения списка файлов.
4.  **Readdir:** Прочитать следующую запись (имя файла) в папке.
5.  **Rename:** Переименовать (как файл).
6.  **Link (Hard Link, Жесткая ссылка):**
    *   Позволяет одному файлу лежать в нескольких папках одновременно.
    *   Это **не копия**. Это две записи в директориях, указывающие на одни и те же данные на диске.
    *   Если удалить один линк, файл остается (пока счетчик ссылок > 0).
7.  **Unlink:** Удалить запись из директории. В UNIX удаление файла — это именно `unlink`. Если ссылок больше нет, данные стираются.
8.  **Symbolic Link (Soft Link, Символическая ссылка / Ярлык):**
    *   Маленький файл, внутри которого написан **путь** к другому файлу.
    *   **Плюсы:** Может указывать на другой диск или компьютер (Hard Link не может).
    *   **Минусы:** Медленнее (нужно прочитать путь и найти заново). Если удалить оригинал, ссылка станет битой.
