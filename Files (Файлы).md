**Файл** — это механизм абстракции. Он позволяет хранить информацию на диске и читать её, не забивая голову тем, как работают головки диска или сектора SSD.

#### 4.1.1 File Naming (Именование файлов)
Когда процесс создает файл, он дает ему имя. Правила именования зависят от ОС:
*   **Длина:** Раньше (MS-DOS) было 8 символов + 3 расширение. Сейчас (современные Windows, Linux) — до 255 символов.
*   **Регистр (Case Sensitivity):**
    *   **UNIX/Linux:** Регистрозависимые. `Maria`, `maria` и `MARIA` — это три разных файла.
    *   **Windows (MS-DOS/FAT):** Регистронезависимые. Для системы это одно и то же имя.
*   **Расширения (Extensions):** Часть имени после точки (например, `.c`, `.txt`).
    *   **В UNIX:** Это просто условность (convention). ОС плевать, как ты назвал файл, хоть `data.txt.zip.tmp`.
    *   **В Windows:** Это имеет смысл. ОС привязывает расширение к программе (кликнул по `.docx` — открылся Word).

#### 4.1.2 File Structure (Структура файла)
Как организованы данные внутри файла? Есть три основных подхода:

1.  **Byte Sequence (Поток байтов):**
    *   ОС видит файл как просто кучу байтов без структуры.
    *   Смысл данным придает только программа пользователя.
    *   **Где используется:** UNIX, Windows, MacOS. Это самый гибкий и популярный вариант сегодня.
2.  **Record Sequence (Поток записей):**
    *   Файл состоит из записей фиксированной длины (например, 80 байт — наследство перфокарт).
    *   Операции чтения/записи идут строго по одной записи.
    *   **Где используется:** Древние мейнфреймы.
3.  **Tree (Дерево):**
    *   Записи разной длины, у каждой есть **ключ**.
    *   Файл — это дерево, отсортированное по ключу для быстрого поиска.
    *   **Где используется:** Мейнфреймы для коммерческой обработки данных.

#### 4.1.3 File Types (Типы файлов)
В UNIX и Windows файлы делятся на:
1.  **Regular files (Обычные файлы):** Содержат данные пользователя.
    *   **ASCII (Текстовые):** Можно читать глазами, редактировать в блокноте. Удобны для пайплайнов в UNIX.
    *   **Binary (Бинарные):** Внутренняя структура понятна только программе. При печати — мусор. Обычно имеют заголовок с метаданными.
    *   **Magic Number:** Бинарные исполняемые файлы часто начинаются с "магического числа" — уникальной сигнатуры, по которой ОС понимает, что этот файл можно запустить.
2.  **Directories (Директории/Каталоги):** Системные файлы, поддерживающие структуру ФС.
3.  **Special files (Специальные файлы):**
    *   *Character special files:* Для ввода/вывода (терминалы, принтеры).
    *   *Block special files:* Для дисков.

#### 4.1.4 File Access (Доступ к файлам)
1.  **Sequential Access (Последовательный):** Читаем байты по порядку с начала. Нельзя прыгнуть в середину (наследие магнитных лент).
2.  **Random Access (Произвольный):** Можно читать байты в любом порядке или по ключу.
    *   Критично для баз данных.
    *   Реализуется с помощью операции **`seek`** (переместить курсор чтения), за которой следует `read`.

#### 4.1.5 File Attributes (Атрибуты файла)
Файл — это не только имя и данные. У него есть **Metadata (Метаданные)**.
Список атрибутов зависит от ОС, но основные:
*   **Protection:** Кто может читать/писать/исполнять.
*   **Flags:** Скрытый, Системный, Архивный (для бэкапов), Только чтение.
*   **Timestamps:** Время создания, последнего доступа, последней модификации.
*   **Size:** Текущий размер файла.


#### 4.1.6 File Operations (Операции с файлами)
Самые распространенные системные вызовы для работы с файлами:
1.  **Create:** Создать пустой файл, задать атрибуты.
2.  **Delete:** Удалить и освободить место.
3.  **Open:** Загрузить атрибуты и список адресов диска в память для быстрого доступа.
4.  **Close:** Освободить ресурсы, сбросить буферы на диск.
5.  **Read / Write:** Чтение и запись данных (обычно через буфер).
6.  **Seek:** Переместить указатель текущей позиции (для Random Access).
7.  **Get/Set attributes:** Чтение/изменение метаданных.

#### 4.1.7 Example Program (Пример программы)
Таненбаум приводит пример простой утилиты `copyfile` на C.
**Суть работы:**
1.  `open` (исходный файл) и `creat` (файл назначения). Получаем **File Descriptors** (маленькие целые числа-идентификаторы).
2.  Цикл `while(true)`:
    *   `read(in_fd, buffer, 4096)` — читаем блок данных (4 КБ) в буфер.
    *   Если прочитано 0 байт — выход из цикла (конец файла).
    *   `write(out_fd, buffer, count)` — пишем данные из буфера в целевой файл.
3.  `close` обоих файлов.

Важно: Чтение и запись блоками (например, по 4096 байт) намного эффективнее побайтового копирования.
