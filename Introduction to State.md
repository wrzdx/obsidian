### Суть в двух словах
**State (Состояние)** — это "память" компонента. Это данные, которые **меняются** со временем (от кликов, таймеров, API) и заставляют React перерисовать (re-render) интерфейс.

### Ключевые концепции

*   **Почему не обычные переменные (`let`)?**
    1.  Изменение локальной переменной **не запускает** рендер (React не видит изменений).
    2.  При повторном рендере локальные переменные **сбрасываются** (инициализируются заново).
*   **Хук `useState`**: Сообщает React: "Следи за этой переменной. Если она изменится — перерисуй компонент".
*   **Изоляция**: Состояние локально для каждого *экземпляра* компонента. Если отрендерить одну и ту же кнопку дважды, у каждой будет свой независимый счетчик.

### Синтаксис и примеры кода

**1. Подключение и инициализация:**
```jsx
import { useState } from 'react';

function Counter() {
  // [текущее_значение, функция_для_изменения] = useState(начальное_значение)
  const [count, setCount] = useState(0); 

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicked {count} times
    </button>
  );
}
```

**2. Как это работает:**
1.  При первом запуске `count` = `0`.
2.  Ты жмешь кнопку -> вызывается `setCount(1)`.
3.  React понимает, что данные изменились, и запускает функцию `Counter` заново.
4.  В этот раз `useState(0)` вернет уже `1` (React "запомнил" новое значение).

### Assignment Highlights (Важное из документации)

1.  **State is a Snapshot (Снимок)**:
    *   `count` — это константа внутри *конкретного* рендера.
    *   Когда ты вызываешь `setCount(count + 1)`, значение изменится только при **следующем** рендере.
    *   *Пример:* Если вызвать `setCount(count + 1)` три раза подряд в одной функции, он увеличится только на 1, потому что во всех трех вызовах берется "старый" снимок `count`.
2.  **`const` для state**:
    *   Мы используем `const [count, setCount]`, хотя значение меняется. Почему? Потому что `setCount` не меняет переменную `count` в текущем выполнении функции, он заставляет React *пересоздать* компонент с новой константой `count`.

### Best Practices / Ошибки

*   ❌ **Ошибка**: Пытаться менять стейт напрямую: `count = 5`. Это не сработает и не обновит UI. Всегда используй сеттер: `setCount(5)`.
*   ✅ **Именование**: Всегда используй паттерн `[noun, setNoun]`.
    *   `[isEnabled, setIsEnabled]`
    *   `[user, setUser]`
*   ✅ **Props vs State**:
    *   **Props**: Передаются от родителя, неизменяемы (как аргументы функции).
    *   **State**: Живет внутри компонента, изменяем (как личная память).
