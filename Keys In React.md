### Суть в двух словах
**Keys (Ключи)** — это специальные строковые атрибуты, которые нужны React, чтобы отслеживать элементы в списках. Они позволяют обновлять интерфейс точечно и эффективно, не перерисовывая весь список при малейшем изменении.

### Ключевые концепции

*   **Идентификация**: Ключ говорит React: "Этот конкретный `<div>` соответствует вот этому объекту данных".
*   **Reconciliation (Сверка)**: Когда данные меняются, React сравнивает новые ключи со старыми:
    *   Ключ тот же? 👉 Оставляет DOM-элемент (просто обновляет контент внутри).
    *   Ключ исчез? 👉 Удаляет элемент.
    *   Ключ появился? 👉 Создает новый элемент.
*   **Стабильность**: Ключи должны быть постоянными. Они не должны меняться при каждом рендере.

### Синтаксис и примеры

**✅ Правильно (ID из данных):**
```jsx
// Лучший вариант - использовать уникальные ID из базы данных
const todos = [
  { id: 'a1', task: 'Сделать уборку' },
  { id: 'b2', task: 'Выучить React' },
];

return (
  <ul>
    {todos.map((todo) => (
      <li key={todo.id}>{todo.task}</li>
    ))}
  </ul>
);
```

**❌ Плохо (Index как ключ):**
```jsx
// Избегай, если список может сортироваться или изменяться!
{todos.map((todo, index) => (
  <li key={index}>{todo.task}</li>
))}
```

### Assignment Highlights (Важное из ссылок)

1.  **Почему нельзя генерировать ключи на лету?**
    *   *Ошибка:* `key={Math.random()}` или `key={uuid()}` прямо внутри `.map()`.
    *   *Последствие:* При **каждом** рендере ключи будут новые. React решит, что старые элементы удалены, а новые добавлены.
    *   *Итог:* Полная перерисовка DOM (медленно) + потеря фокуса в инпутах и выделения текста.
2.  **Почему `index` — это антипаттерн?**
    *   Если ты удалишь первый элемент списка, все индексы сдвинутся. React подумает, что элементы *поменяли контент*, а не порядок.
    *   Это вызывает баги, если внутри компонентов есть свое состояние (например, введенный текст в `<input>`). Текст останется на месте, хотя элемент списка "уехал".

### Best Practices / Ошибки

*   ✅ **Уникальность**: Ключи должны быть уникальны только **среди соседей** (внутри одного массива). Им не нужно быть уникальными во всем приложении.
*   ✅ **Где ставить ключ**: Всегда вешай `key` на **самый верхний элемент**, который возвращает `.map()`.
    *   Если возвращаешь `<div key={id}>...</div>` — ключ на div.
    *   Если возвращаешь `<MyComponent key={id} />` — ключ на компонент.
    *   Если возвращаешь `<Fragment key={id}>...</Fragment>` — ключ на фрагмент (используй явный импорт `Fragment`, так как сокращенный `<>` ключи не поддерживает).
