# What architecture is used nowadays?

Today, the most commonly used architecture is the **Von Neumann architecture**, which utilizes a single memory space for both instructions and data. However, the **Harvard architecture** is also employed in specific applications like digital signal processing for its ability to access instructions and data simultaneously. Modern CPUs often incorporate elements from both architectures to optimize performance.

# What is the Latency?

**Latency** refers to the delay or time it takes for a system to respond to a request or for data to travel from one point to another. In computing, it often describes the time it takes for data to be retrieved from memory or for a signal to travel between components, affecting overall system performance. Lower latency is generally desirable as it leads to faster response times and improved efficiency.

# What is the Execution time?

**Execution time** is the total time taken by a computer to execute a specific task or program from start to finish. It includes all stages of execution, such as fetching instructions, decoding them, executing operations, and accessing memory. Execution time is an important measure of performance, as shorter execution times indicate faster processing and better efficiency.

# What is the Throughput?

**Throughput** refers to the amount of work or data processed by a system in a given amount of time. In computing, it often measures how many tasks, transactions, or data units are completed in a specific time period, such as instructions per second (IPS) or bits per second (bps). Higher throughput indicates better performance, as it means the system can handle more work efficiently within the same timeframe.

# Latency vs Throughput?

- **Latency** refers to the time delay between a request for data and the moment the data is available. It measures how long it takes for a single operation to complete, indicating responsiveness. Lower latency is desirable for faster responses.

- **Throughput** measures the amount of work done in a specific period of time, indicating how many operations or units of data are processed in that timeframe. Higher throughput signifies better performance, as it reflects the system's ability to handle more tasks concurrently.

# What is the Bandwidth?

**Bandwidth** refers to the maximum rate at which data can be transmitted over a network or communication channel, typically measured in bits per second (bps). It represents the theoretical maximum capacity. In contrast, **throughput** is the actual rate of data transfer achieved in practice, which can be lower than bandwidth due to factors like latency and network congestion. In essence, bandwidth is the potential capacity, while throughput is the real-world usage of that capacity.

# Bandwidth vs Throughput?

**Bandwidth** is the maximum rate at which data can be transmitted over a network or communication channel, measured in bits per second (bps), indicating the theoretical capacity of the connection. In contrast, **throughput** is the actual rate of successful data transfer achieved in practice, which can be lower than bandwidth due to factors like latency, packet loss, and network congestion. Essentially, bandwidth represents the potential capacity, while throughput reflects real-world performance.

# What is the Utilization?

**Utilization** refers to the proportion of the available capacity of a resource that is being used over a specific period. In computing and networking, it often measures how effectively a system's resources (like CPU, memory, or network bandwidth) are being utilized compared to their total capacity. Higher utilization indicates that resources are being used efficiently, while lower utilization may suggest underuse or inefficiencies in resource allocation.
