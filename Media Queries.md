# Syntax

```css
body {
  margin: 24px;
}

@media (max-width: 600px) {
  body {
    margin: 8px;
  }
}
```

На всех экранах которые меньше либо равны 600px будет *margin* 8px, а у остальных 24px.

После `{css}@media` идут опциональные типы медиа (экран, принтер и тд, по умолчанию значение `all`), далее медиа функции (максимальный размер окна, предпочтительный режим, есть ли *hover*). Медиа запросы не чувствительны к регистру.

Существуют дискретные и диапазонные медиа функции. Для диапазонных можно использовать `<=` или `>=` вместо `max-` и `min-` 

```css
@media (min-width: 30em) and (max-width: 50em) { /* … */ } 
@media (30em <= width <= 50em) { /* … */ } 
@media (50em >= width >= 30em) { /* … */ }
```

Можно медиа запросы перечислят через запятую, это будет все равное что использовать `or`
Кроме `or`, есть еще `and`, `not` и `only`.
```css
/* запятая все равно что или */
@media (height >= 680px), screen and (orientation: portrait) {
  /* … */
}
```

`not` применяется ко всем медиа функциям в медиа запросе
```css
/* not только для screen and color */
@media not screen and (color), print and (color) {
  /* … */
}
```

После `not` нужно обернуть скобками запросы состоящие только из медиа фукнций, и нельзя если там есть медиа тип.

Для того чтобы применить `not` только к определенной функции надо заключить и `not` и функция в одни скобки.
```css
@media (not (width > 1000px)) and (color), print and (color) {
  /* … */
}
```

`not` применяется только к элементам справа.

# Common breakpoints

Смартфоны ≤ 500, планшеты ≤ 1000, ноуты и компы ≤ 2000

# Zooming

Зум может влиять на разрешение страницы. Увеличение будет как если у тебя экран был меньше, а уменьшение будет как если твой экран был больше.

# Другие медиа запросы

- Для принтеров
```css
@media print {
  /* print styles go here! */
}
```

- ориентация экрана, соотношение сторон, персональные настройки пользователей (темных/светлых режим, сокращение движения и тд)
