### Суть в двух словах
State в React можно менять **только** через замену всего значения, а не через изменение внутренностей. Для объектов и массивов это означает создание **копий** (Immutability).

### Ключевые концепции

1.  **Мутация = Зло**: React сравнивает состояние по ссылкам в памяти. Если ты сделаешь `user.name = 'Alex'`, ссылка на объект останется старой, и React **не увидит изменений**, а значит, не перерисует интерфейс.
2.  **Spread Syntax (`...`)**: Главный инструмент для копирования данных. Он берет все старые свойства и позволяет перезаписать нужные.
3.  **Batching (Пакетная обработка)**: React "ждет", пока выполнятся все команды `setState` внутри одного обработчика событий, и делает **один** рендер в конце. Это улучшает производительность.

### Синтаксис и примеры (Самое важное)

**1. Объекты (Обновление поля):**
```javascript
const [person, setPerson] = useState({ name: 'Ivan', age: 25, city: 'Moscow' });

// ✅ Правильно: Копируем всё старое (...person), заменяем name
setPerson({ ...person, name: 'Dmitry' });
```

**2. Массивы (Добавление):**
```javascript
// ❌ Нельзя: list.push(newItem)
// ✅ Можно: Создаем новый массив
setList([...list, newItem]); // В конец
setList([newItem, ...list]); // В начало
```

**3. Массивы (Удаление — `filter`):**
```javascript
// Удаляем элемент с id = 5
setList(list.filter(item => item.id !== 5));
```

**4. Массивы (Изменение элемента — `map`):**
```javascript
// Меняем имя у элемента с id = 3
setList(list.map(item => {
  if (item.id === 3) {
    return { ...item, name: 'Updated Name' }; // Возвращаем измененную копию
  }
  return item; // Остальные не трогаем
}));
```

### Assignment Highlights

*   **Глубокая вложенность (Deeply Nested State)**:
    Если у тебя объект в объекте в объекте (`obj.a.b.c`), обновлять его через `...` становится болью:
    ```javascript
    setObj({ ...obj, a: { ...obj.a, b: { ...obj.a.b, c: 'new' } } });
    ```
    *Совет:* Старайся делать структуру стейта плоской (Flat) или используй библиотеки типа **Immer** (они позволяют писать мутирующий код, но под капотом делают копии).

### Best Practices

*   ✅ **Считай данные, а не храни их**: Если у тебя есть `firstName` и `lastName`, не создавай в стейте `fullName`. Просто посчитай `const fullName = firstName + lastName` прямо во время рендера. Это избавит от багов рассинхронизации.
*   ✅ **Используй ID**: При работе с массивами всегда используй уникальные ID для поиска элементов (в `map` и `filter`), а не индексы массива.
