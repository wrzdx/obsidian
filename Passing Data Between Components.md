### Суть в двух словах
В React данные текут **только в одну сторону**: сверху вниз (от родителя к ребенку). Чтобы передать данные обратно (от ребенка к родителю), мы передаем функции.

### Ключевые концепции

1.  **Props (Свойства)**:
    *   Это способ передачи данных от Родителя ➡️ Ребенку.
    *   Похожи на HTML-атрибуты, но могут быть любыми значениями JS (числа, объекты, массивы, функции).
    *   **Props — неизменяемы (Read-only)**. Ребенок не может их менять.
2.  **Деструктуризация (Destructuring)**:
    *   Вместо того чтобы везде писать `props.title`, `props.color`, мы "распаковываем" объект сразу в аргументах функции.
3.  **Functions as Props (Функции как пропсы)**:
    *   Единственный способ для Ребенка "пообщаться" с Родителем.
    *   Родитель передает функцию (как коллбэк). Ребенок вызывает ее, когда происходит событие (клик, ввод текста).

### Синтаксис и примеры

**1. Передача данных (Parent ➡️ Child):**
```jsx
// Родитель (Parent.jsx)
function App() {
  const userName = "Alex";
  
  // Передаем строку и число
  return <Profile name={userName} age={25} />;
}
```

**2. Прием данных (Child):**
*Старый способ (без деструктуризации):*
```jsx
function Profile(props) {
  return <h1>{props.name}</h1>;
}
```
*✅ Современный способ (с деструктуризацией):*
```jsx
// Распаковка прямо в аргументах
function Profile({ name, age }) {
  return (
    <div>
      <h1>Name: {name}</h1>
      <p>Age: {age}</p>
    </div>
  );
}
```

**3. Обратная связь (Child ➡️ Parent через функции):**
```jsx
// Родитель
function App() {
  const handleClick = () => alert("Button clicked inside Child!");

  // Передаем ФУНКЦИЮ как пропс
  return <Button onButtonClick={handleClick} />;
}

// Ребенок
function Button({ onButtonClick }) {
  // Вызываем функцию, которую дал родитель
  return <button onClick={onButtonClick}>Click me</button>;
}
```

### Assignment Highlights (Важные нюансы)

*   **Значения по умолчанию (Default Props)**:
    Можно задать значение пропса, если родитель забыл его передать, прямо при деструктуризации:
    ```jsx
    function Button({ color = 'blue', text }) { 
      // Если color не передали, он будет 'blue'
      return <button className={color}>{text}</button>; 
    }
    ```
*   **Спред пропсов (Prop Spreading)**:
    Если нужно передать много пропсов дальше "сквозь" компонент, используют синтаксис spread (`...`):
    ```jsx
    function Profile(props) {
      return <Avatar {...props} />; // Передает всё, что пришло в Profile, дальше в Avatar
    }
    ```
    *Использовать с осторожностью, чтобы не передать лишнего мусора.*

### Best Practices / Ошибки

*   ✅ **Именуй функции-пропсы понятно**:
    *   Проп (атрибут): `onClick`, `onSave`, `handleLogin`.
    *   Сама функция: `handleClick`, `saveData`, `loginUser`.
    *   Пример: `<Button onClick={handleClick} />`.
*   ❌ **Не изменяй пропсы**: Никогда не пиши `props.text = "New text"`. Это работать не будет (см. предыдущий урок).
*   ✅ **Чистые функции**: Старайся делать компоненты чистыми — результат рендера должен зависеть только от пропсов. Тот же пропс -> тот же внешний вид.
