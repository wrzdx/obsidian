**Главная идея:**  
Процесс — ключевая абстракция операционной системы. Всё в ОС строится вокруг неё.

**Что такое процесс:**  
Абстракция _выполняющейся программы_. Позволяет операционной системе представлять выполнение как отдельный, изолированный поток управления.

**Зачем нужен процесс:**  
Процессы создают иллюзию параллельности даже на одном CPU. Один реальный процессор превращается в множество виртуальных «процессоров», по одному на процесс.

**Почему это важно:**  
Современные системы всегда делают много дел одновременно:  
• серверы обрабатывают десятки/сотни запросов, многие из которых ждут медленный диск;  
• персональные компьютеры запускают процессы в фоне — антивирус, почтовые демоны, индексацию файлов, программы пользователя;  
• телефоны тоже работают с множеством одновременно активных процессов.

**Как создаётся иллюзия параллельности:**  
CPU быстро переключается между процессами — каждые десятки или сотни миллисекунд.  
Фактически в каждый момент времени выполняется один процесс на каждом ядре, но за секунду ядро успевает обработать множество процессов, и человек воспринимает всё как параллельную работу.

**Псевдопараллелизм vs. аппаратный параллелизм:**  
• _Псевдопараллелизм:_ один CPU переключает процессы и создаёт иллюзию одновременности.  
• _Настоящий параллелизм:_ несколько CPU (или ядер) выполняют разные процессы буквально одновременно.

**Почему нужна модель процессов:**  
Одновременные активности сложно отслеживать напрямую. Процессы дают удобную абстракцию последовательного выполнения, на которую накладывается механизм переключения — так проектировать ОС проще.


## The Process Model

ОС организует всё выполняемое ПО как набор _процессов_.

**Процесс** — это выполняющийся экземпляр программы, включающий:  
• текущее состояние (program counter),  
• значения регистров,  
• переменные.

Каждый процесс концептуально имеет свой _виртуальный CPU_.  
На деле реальный CPU просто быстро переключается между процессами — это и есть **multiprogramming**.

Процессор хранит логический program counter процесса и подменяет его при переключении. В любой момент времени выполняется только один процесс (на одном ядре), но за долгий интервал все процессы продвигаются.

В тексте предполагается один CPU, но идея такая же и для многоядерных систем — _одно ядро = один одновременно выполняющийся процесс_.

Из-за частых переключений скорость продвижения процесса непредсказуема. Программы не должны полагаться на точное время выполнения своих действий. Для задач с жёсткими временными ограничениями (real-time) нужны отдельные механизмы обеспечения сроков.

**Различие между программой и процессом:**  
• _Программа_ — пассивный код на диске.  
• _Процесс_ — выполняющаяся активность: код + данные + текущее состояние.

Если одна и та же программа запущена дважды, это два разных процесса с независимыми состояниями. Код может быть общим, но процессы остаются отдельными сущностями.

## Process Creation

ОС должна уметь создавать процессы во время работы. В разных системах это делается по-разному, но есть четыре типичных случая, когда создаются процессы:

1. при запуске ОС;
    
2. когда работающий процесс вызывает системный вызов для создания нового;
    
3. когда пользователь запускает программу;
    
4. при запуске batch-заданий на больших системах.

При загрузке ОС создаютсяForeground-процессы (работающие с пользователем) иBackground-процессы — _daemons_. Демоны обрабатывают почту, веб-запросы, печать и другие фоновые задачи. Их может быть десятки.

После загрузки новые процессы создаются, когда:  
• работающий процесс создает дочерние процессы, чтобы распараллелить работу;  
• пользователь запускает программу командой или кликом;  
• в batch-системе ОС решает, что может обработать очередное задание.

Во всех случаях создание нового процесса технически происходит через системный вызов, который выполняет существующий процесс.

**UNIX:** используется один системный вызов — `fork`.  
Он создает _клон_ процесса: копию памяти, окружения и открытых файлов. Затем обычно дочерний процесс вызывает `execve` для загрузки новой программы.  
Разделение на “fork → execve” нужно для корректной перенастройки файловых дескрипторов (например, перенаправление stdout).

**Windows:** использует функцию `CreateProcess`.  
Она одновременно создает процесс и загружает в него нужную программу. У неё много параметров: путь к программе, аргументы командной строки, атрибуты, права, параметры окна, приоритет и т. д.

**Общее для UNIX и Windows:**  
• родитель и потомок имеют независимые адресные пространства;  
• изменение памяти в одном процессе не видно в другом;  
• текст программы может быть общим, но _записываемая_ память не разделяется;  
• иногда используется “copy-on-write”: копирование памяти происходит только при изменении данных.
