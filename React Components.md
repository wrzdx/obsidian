### Суть в двух словах
Компоненты — это строительные блоки React-приложения. Технически это просто **JavaScript-функции**, которые возвращают разметку (JSX), которую React превращает в HTML.

### Ключевые концепции

*   **Компонент = Функция**: React-компонент — это JS-функция, имя которой **обязательно** начинается с заглавной буквы (PascalCase).
*   **Декларативный стиль**: Вы описываете, *как* интерфейс должен выглядеть (пишете JSX), а не *как* его менять (как в императивном `document.createElement`).
*   **Вложенность (Nesting)**: Компоненты могут рендерить другие компоненты внутри себя, создавая дерево (как HTML внутри HTML).
*   **Корневой компонент**: Обычно есть один главный компонент (часто `App`), в который вложены все остальные.

### Синтаксис и примеры кода

**1. Создание компонента:**
```jsx
// Имя функции с Большой Буквы!
function MyComponent() {
  return <h1>Hello World</h1>;
}

export default MyComponent;
```

**2. Использование компонента:**
Мы используем функцию как HTML-тег:
```jsx
import MyComponent from './MyComponent';

function App() {
  return (
    <div>
      <MyComponent />  {/* Self-closing tag, если нет детей */}
      <MyComponent></MyComponent>
    </div>
  );
}
```

### Assignment Highlights (Важное из JSX)

Ссылки в уроке ведут к правилам JSX (Writing Markup with JSX):
1.  **Возврат одного родителя**: Компонент не может вернуть два соседних тега (например, два `<h1>`). Их нужно обернуть в `<div>` или **Фрагмент** (`<>...</>`), чтобы в DOM не летел лишний мусор.
    ```jsx
    // Нельзя: return <h1>Hi</h1><h2>Bye</h2>;
    // Можно:
    return (
      <>
        <h1>Hi</h1>
        <h2>Bye</h2>
      </>
    );
    ```
2.  **Закрытие тегов**: Все теги должны быть закрыты. `<br />`, `<img />` (в HTML можно было `<br>`, в JSX — ошибка).
3.  **camelCase**: Атрибуты пишутся в camelCase. `onclick` -> `onClick`, `stroke-width` -> `strokeWidth`.
4.  **Специальные атрибуты**:
    *   `class` ➡️ **`className`** (так как `class` зарезервировано в JS).
    *   `for` ➡️ **`htmlFor`** (для label).

### Best Practices / Ошибки

*   ❌ **Ошибка**: Объявлять компонент *внутри* другого компонента. Это убивает производительность (React будет пересоздавать его при каждом рендере). Выноси их наружу.
    ```jsx
    // ТАК НЕЛЬЗЯ
    function App() {
      function BadComponent() { return <div />; } 
      return <BadComponent />;
    }
    ```
*   ✅ **Круглые скобки в return**: Если JSX занимает несколько строк, всегда оборачивай его в `()`, иначе JS может автоматически подставить `;` после `return`, и ничего не вернется.
*   ✅ **Именование**: Всегда называй компоненты с **Заглавной** буквы (`Button`, а не `button`), иначе React подумает, что это встроенный HTML-тег.
