### Суть в двух словах
React не добавляет специальный синтаксис для циклов или условий (как `v-if` или `ng-for`). Всё делается через нативный **JavaScript** (`if`, `&&`, `.map()`).

### Ключевые концепции

1.  **Условный рендеринг (Conditional Rendering)**:
    *   Используем логику JS, чтобы показать или скрыть компонент.
    *   React игнорирует (не рендерит) `null`, `undefined` и `false`.
2.  **Рендеринг списков (Rendering Lists)**:
    *   Используем метод массива **`.map()`**, чтобы превратить массив данных в массив JSX-элементов.
    *   Метод `.forEach()` не подходит, так как он ничего не возвращает, а нам нужно вернуть массив элементов.
3.  **Keys (Ключи)**:
    *   Уникальный идентификатор (`key`) помогает React понять, какой элемент списка изменился, добавился или исчез.

### Синтаксис и примеры кода

**1. Условия (Внутри JSX):**

*   **Тернарный оператор** (Если/Иначе):
    ```jsx
    {isLoggedIn ? <UserPanel /> : <LoginButton />}
    ```
*   **Логическое "И"** (Показать, если true, иначе ничего):
    ```jsx
    {hasError && <ErrorMessage />}
    ```
    *Работает по принципу Short Circuit: если левая часть true, возвращается правая.*

**2. Списки (.map):**

```jsx
const fruits = ['Apple', 'Banana', 'Orange'];

return (
  <ul>
    {fruits.map((fruit) => (
      <li key={fruit}>{fruit}</li> // key обязателен!
    ))}
  </ul>
);
```

### Assignment Highlights (Важные нюансы)

1.  **Проблема нуля (`0`) и `&&`**:
    *   В JS число `0` — это "falsy", но React **рендерит** число 0.
    *   *Ошибка:* `itemsCount && <Cart />`. Если `itemsCount` равен `0`, в браузере отобразится цифра `0`, а не пустота.
    *   *Решение:* `itemsCount > 0 && <Cart />` или `!!itemsCount && <Cart />`.
2.  **Ключи (`key`)**:
    *   Ключ должен быть **уникальным среди соседей**.
    *   Ключ **не должен меняться** при перерисовках. Генерация ключа "на лету" (`key={Math.random()}`) запрещена — это заставит React пересоздавать DOM-элементы каждый раз, сбрасывая фокус и инпуты.

### Best Practices / Ошибки

*   ✅ **Ключи на верхнем уровне**: Если ты выносишь элемент списка в отдельный компонент, `key` должен быть **у вызова компонента**, а не внутри него.
    ```jsx
    // Правильно:
    items.map(item => <ListItem key={item.id} data={item} />)
    ```
*   ❌ **Использование `index` как ключа**: Избегай `key={index}`, если список может меняться (сортировка, удаление). Это приведет к багам с состоянием. Используй ID из базы данных.
*   ✅ **Чистота JSX**: Если условие слишком сложное, не пиши его внутри `return (...)`. Вынесли логику в переменную перед `return` или в отдельную функцию.
