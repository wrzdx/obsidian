# Основы

Самая частая проблема – это сохранение пропорций картинки, решение – не объявлять обы `width` и `height`, в таком случае один из них будет со значением `auto`, и будем сохрянять изначальные пропорции картинки.

# background-size, background-position and object-fit

- `background-size` + `background-position` предоставляют довольно много контроля над отображением изображения. Работают они на элементах у которых есть *background image* (для `img` тегов не работает). 
  Значения для `background-size` – `cover`, `contain`, `auto`, одно значение для ширины, если два то второе для высоты.
  Значения для `background-position` – `top`, `right`, `bottom`,`left`, `center`, первое значение отступ слева, второе отступ сверху, если второе не указано то 50%. Если третье значение определено, то оно отнимается от первое (например bottom right 50px, будет отсчитано 50px снизу), четвертое значение отнимается от второго.

- `object-fit` – как `background-size`, но работает для `img` тегов.
  Значения могут быть `fill` (по умолчанию), `contain`, `cover`, `none`, `scale-down`.

# Адаптивное изменение изображения

- Проблема сохранения по разному обрезанных версий изображения в зависимости от макета называется **art direction problem**.

- Проблема загрузки одного изображения с разными разрешениями в зависимости от размера экрана устройства называется **resolution switching problem**.

## Resolution Switching

### Different sizes

Стандартный `img` позволяет толко указывать браузеру на истноник изображения:
```html
<img src="elva-fairy-800w.jpg" alt="Elva dressed as a fairy" />
``` 

Однако мы можем использовать два атрибута – `srcset` и `sizes` – для предоставления дополнительных источников и подсказок для правильного выбора между ними. 
```html
<img
  srcset="elva-fairy-480w.jpg 480w, elva-fairy-800w.jpg 800w"
  sizes="(width <= 600px) 480px,
         800px"
  src="elva-fairy-800w.jpg"
  alt="Elva dressed as a fairy" />
```
 
- `srcset` – объявляет список истоников – *<источник> <размер в пикселях>w*
- `sizes` – объявляет медиа запросы – *<условие> <размер изорбажения><любая мера длины кроме процентов>*, в последнем запросе условие можно не писать и оставить лишь какое то значение, далее . 
  Если размера изображения нет в слотах `srcset`, то выбирается первый размером больше указанного и масштабируется до нужного размера.
  Можно использовать для гибкости `{html}calc()`:
```html
<img 
  ...  
  sizes="
    (max-width: 500px) calc(100vw - 2rem), 
    (max-width: 700px) calc(100vw - 6rem),
    calc(100vw - 9rem - 200px)
  "
  /* Но для большей точности можно использовать Horseshoes & Hand Grenades Method™ */
 
/>
```

  Для тестирования можно проверять в вкладке *Network Monitor* какое изображение загружается.
- `src` – лучше устанавливать для обратной совместимости со старыми браузерами которые не подреживают вышесказанные фичи. 
  

### Same size, different resolutions

Для low-resolution экранов, картинка будет загружаться с низким качеством и соответственно с легким весом, в то время как для high-resolution экранов с высоким качеством и более тяжелым весом.
```html
<img
  srcset="elva-fairy-320w.jpg, elva-fairy-480w.jpg 1.5x, elva-fairy-640w.jpg 2x"
  src="elva-fairy-640w.jpg"
  alt="Elva dressed as a fairy" />
```

Здесь мы не указываем `sizes` так как браузер сам будет выбирать в зависимости от разрешения экрана пользователя, указывается сколько *device pixels* приходится на один *css pixel* в `srcset`.


## Art direction

Для реализации логики, когда для разных размеров экрана используем разные изображения, мы используем `picture` тег.
```html
<picture>
  <source media="(width < 800px)" srcset="elva-480w-close-portrait.jpg" />
  <source media="(width >= 800px)" srcset="elva-800w.jpg" />
  <img src="elva-800w.jpg" alt="Chris standing up holding his daughter Elva" />
</picture>
```

`picture` элемент является оберткой для нескольких `source` элементов, которые предоставляют источники для разных изображений, и в конце `img` элемент как значение по умолчанию.

- `source` содержит `media` атрибут, в котором находится медиа запрос, и `srcset` для указания одного или нескольких источникох изображения для разных разрешений экрана пользователя.
