### Суть в двух словах
Для создания React-приложений больше не используется `create-react-app`. Современный стандарт — сборщик **Vite**, который работает в разы быстрее, легче и не тянет за собой устаревший багаж Webpack.

### Ключевые концепции

*   **Create React App (CRA) — RIP 💀**:
    *   Раньше был стандартом (с 2016 года).
    *   Официально **deprecated** (устарел) в 2023.
    *   Медленный на больших проектах, так как использовал Webpack и пересобирал всё приложение при каждом изменении.
*   **Vite ⚡**:
    *   Новый стандарт для старта React проектов (рекомендация The Odin Project и React docs).
    *   Использует **ES Modules** браузера, поэтому сервер запускается мгновенно.
    *   **HMR (Hot Module Replacement)** работает моментально, сохраняя состояние приложения (state) при редактировании кода.

### Синтаксис и команды (Terminal)

**1. Создание нового проекта:**
```bash
npm create vite@latest my-first-react-app
```
*Вас спросят:*
*   *Framework:* Выбрать `React`.
*   *Variant:* Выбрать `JavaScript` (или `JavaScript + SWC` для еще большей скорости компиляции).

**2. Запуск проекта:**
После создания папки нужно зайти в нее, установить зависимости и запустить сервер:
```bash
cd my-first-react-app
npm install      # Установка пакетов (node_modules)
npm run dev      # Запуск сервера разработки
```
*Сервер обычно стартует на `http://localhost:5173/`*

### Структура проекта (Vite vs CRA)

Vite немного меняет привычную структуру файлов:

*   **`index.html`** — Теперь лежит в **корне проекта** (root), а не в папке `public`. Это точка входа; Vite обрабатывает его и подключает JS-скрипты.
*   **`src/main.jsx`** (или `index.jsx`) — JavaScript точка входа. Здесь React "вмонтируется" в DOM элемент с `id="root"`.
    ```jsx
    import React from 'react'
    import ReactDOM from 'react-dom/client'
    import App from './App.jsx'
    import './index.css'

    ReactDOM.createRoot(document.getElementById('root')).render(
      <React.StrictMode>
        <App />
      </React.StrictMode>,
    )
    ```
*   **`.jsx` extension** — Vite требует, чтобы файлы, содержащие JSX, имели расширение `.jsx`, а не `.js` (в отличие от старого Webpack, который это прощал).

### Assignment Highlights (Важное из ссылок)

1.  **Почему CRA умер?**
    *   CRA скрывал конфигурацию Webpack, которую было почти невозможно изменить без команды `eject` (которая ломала возможность обновлений). Vite дает прозрачный конфиг `vite.config.js`.
2.  **Strict Mode (`<React.StrictMode>`)**:
    *   В режиме разработки (`dev`) намеренно **рендерит компоненты дважды**.
    *   *Зачем?* Чтобы выявлять побочные эффекты (side effects) и устаревшие методы.
    *   *Влияние:* Если видишь в консоли два `console.log` подряд — это нормально, в продакшене этого не будет.

### Best Practices / Ошибки

*   ❌ **Не используй** `npx create-react-app` для новых проектов.
*   ✅ **Всегда делай `npm install`** сразу после создания папки (Vite не делает это автоматически, чтобы вы могли выбрать менеджер пакетов сами).
*   ✅ **Расширения файлов**: Если в файле есть `<Component />`, называй его `Component.jsx`. Если только логика — `.js`.
*   ✅ **Зависимости**: React обновляется часто. Если берешь старый туториал, версии библиотек могут конфликтовать. Vite обычно ставит самые свежие версии.
