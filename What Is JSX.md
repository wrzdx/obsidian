### Суть в двух словах
JSX (JavaScript XML) — это **синтаксический сахар** для JavaScript. Он позволяет писать код, визуально похожий на HTML, внутри JS-файлов, но "под капотом" это превращается в обычные JavaScript-объекты.

### Ключевые концепции

*   **Это не HTML**: Браузеры не понимают JSX. Сборщик (Vite/Babel) компилирует его в вызовы функций React (например, `_jsx(...)` или старый `React.createElement`), которые создают виртуальный DOM.
*   **"Окно" в JavaScript**: Фигурные скобки `{}` позволяют вставлять любые валидные JS-выражения прямо в разметку.
*   **Разделение ответственности**: Вместо разделения технологий (HTML отдельно, JS отдельно), React разделяет *ответственность* (компонент содержит и логику, и вид).

### Синтаксис и примеры кода

**1. Вставка переменных и выражений:**
```jsx
const name = "Alex";
const element = <h1>Hello, {name}</h1>; // Использование переменной

const sum = <p>2 + 2 = {2 + 2}</p>; // Вычисление выражения
```

**2. Атрибуты (Строки vs Переменные):**
```jsx
// 1. Строковый литерал (нужны кавычки)
<img src="avatar.png" className="photo" />

// 2. JavaScript переменная (кавычки НЕ нужны!)
<img src={user.avatarUrl} className={myClass} />
```

**3. Inline Styles (Объект внутри скобок):**
Стили в React пишутся как объекты, поэтому скобки двойные: одни для JS, вторые для объекта.
```jsx
<div style={{ backgroundColor: 'black', color: 'white' }}>
  Styled Text
</div>
```

### Assignment Highlights (Важное из документации)

1.  **Expressions vs Statements**: Внутри `{}` можно писать только **выражения** (expressions) — то, что возвращает значение (математика, тернарные операторы, методы массивов типа `.map()`).
    *   *Нельзя* писать инструкции (statements): `if`, `for`, `switch`, объявление переменных (`const x = 1`).
2.  **`undefined` и `null`**: Если выражение внутри `{}` возвращает `null`, `undefined` или `false`, React ничего не рендерит (пустое место). Это удобно для условного рендера.

### Best Practices / Ошибки

*   ❌ **Ошибка**: Использование кавычек вокруг фигурных скобок в атрибутах.
    *   `src="{avatar}"` — React воспримет это как строку "{avatar}".
    *   Правильно: `src={avatar}`.
*   ✅ **Условный рендеринг**: Так как `if` внутри JSX не работает, используй тернарный оператор:
    ```jsx
    <div>
      {isLoggedIn ? <AdminPanel /> : <LoginForm />}
    </div>
    ```
