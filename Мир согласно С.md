**1.8.1 The C Language**

C используется для ОС, потому что:  
• есть указатели и полный контроль над памятью;  
• нет сборщика мусора → выполнение предсказуемо;  
• всё управляется вручную: выделение, освобождение, структуры данных.

Особенности:  
• Примитивы: int/short/long, char, float/double.  
• Есть массивы, структуры, объединения (union).  
• Управляющие конструкции как в Java: if, switch, for, while.  
• Указатели дают доступ к адресам и позволяют работать с raw memory.  
• Нет встроенных строк, объектов, пакетов, потоков, GC, типобезопасности.  
• Доступ к памяти прямой, поэтому ошибки легко допустить.

Именно контроль над памятью делает C подходящим для низкоуровневых систем.

**1.8.2 Header Files**

Программа ОС — это множество .c и .h файлов.

• .h содержит объявления функций, константы, макросы, структуры.  
• \#define используется для объявления констант и макросов.  
• Есть условная компиляция (#ifdef … \#endif) — включает код в зависимости от архитектуры или конфигурации.  
• .c файлы включают нужные .h через \#include.  
• В крупных проектах много общих header'ов.

**1.8.3 Large Programming Projects**

Процесс сборки:

1. C preprocessor обрабатывает \#include, макросы и условную компиляцию.
    
2. C compiler превращает каждый .c в объектный файл (.o).
    
3. make отслеживает зависимости и компилирует только изменённые файлы.
    
4. linker объединяет все .o и библиотеки в один исполняемый файл (a.out).
    
5. Любое крупное приложение, не только ОС, строится так же.

**1.8.4 The Model of Run Time**

Исполнимый файл состоит из сегментов:  
• text — машинный код, неизменяемый;  
• data — глобальные данные, может расти;  
• stack — вызываемые функции, растёт и сокращается по мере исполнения.

Размещение обычно: text внизу, data над ним, stack — сверху вниз.  
ОС выполняется напрямую железом, без интерпретации и без JIT.
