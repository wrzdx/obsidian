# CPUs (Central Processing Units)
Основной цикл процессора состоит из извлечения первой инструкции из памяти, декодирование его, определения операции и операндов, и выпонение его, затем последующие циклы.

Каждый процессор имеет определенный **набор инстукций** которые он может исполнять. 

Рядом с процессором находятся **регистры** для хранения переменных. Наборы иструкций часто включают операции для загрузки **машинного слова** с памяти в регистр и для записи слова с регистра в память.

Существуют также особые регистры видимые программисту. 

Одним из них является счетчик команд (**program counter**), который содержит в себе адрес памяти следующей инструкции. 

Другим таким регистром является указатель стека (**stack pointer**) который указывает на вершину текущего стека в памяти. Стек хранит один фрейм для каждой процедуры которая была вызвана но не завершена. Он сохраняет вводные значения, локальные переменные и временные значения которые не содерждатся в регистрах.

Еще один особый регистр это **PSW (Program Status Word)**. Содержит различную мета-информацию программы: приоритет выполнения, режим исполнения (режим ядра *kernal mode* или пользователя *user mode*) и др. 

Эти регистры нужны чтобы процессор мог отложить текущую программу, а затем когда снова придет его черед, снова продолжить на том же месте где остановился. 

По факту, мы различили только между собой **архитектуру** и **микро-архитектуру**. Архитектура состоит из всего что видно для програмному обеспечению – инструкции и регистры. А микро-архитектура включает в себя реализацию архитектуры.

Для увелечения производительности придумали идею конвейеризации (**pipelining**), когда процессор исполняет несколько инструкций одновременно:

![800](Pasted%20image%2020251105215951.png)

Более продвинутой техникой конвейеризации является *Superscalar CPU*, в котором содержится несколько устройств для выполнения различных операций.

Как раньше упоминалось, *kernel mode* дает полный контроль над аппаратным обеспечением, в то время как *user mode* может выполнять лишь ограниченное набор команд. В каком режиме работает программа написано на *PSW*. 

Пользовательские программы работают в режиме пользователя (_user mode_). Для выполнения привилегированных операций (таких как ввод-вывод) они должны делать системные вызовы (**system calls**). Такой вызов генерирует прерывание (**trap**), которое переключает режим выполнения на режим ядра (_kernel mode_). После этого управление передается операционной системе, и она, имея необходимые привилегии, выполняет запрошенную операцию. После ее завершения меняеют режим и возвращает контроль вызвавшей команде.

Кроме системных вызовов, прерывания могут генерироваться аппаратным обеспечением. Большинство из них вызываются чтобы предупредить об исключительных ситуациях, таких как деление на ноль. В подобных случаях управление передается ОС. Пользовательские программы могут попросить передать управление им, в определенных случаях.

## Многопоточные и многоядерные процессоры

Согласно закону Мура, количество транзисторов на процессоре удваивается каждые 18 месяцев. Однако бесконечно транзисторов располагать не получится, поэтому ввели понятние многопоточность (**multithreading** or **hyperthreading**). Его суть заключалась в том, что процессор теперь мог хранить состояние нескольких потоков, между которыми можно быстро переключаться. Потоки это как облегченные процессы. Многопоточность не дает настоящий параллелизм. 

В настоящий момент, множество процессоров имеют несколько **ядер**, каждый из которых представляет собой мини процессор. Использование нескольких ядер требует естественно многопроцессорных ОС.

![800](Pasted%20image%2020251105224625.png)

Современные **GPU** (**Graphics Processing Unit**) состоят из тясяч крошечных ядер. Они хороши для маленьких параллельных вычислений, как рендеринг. Они не так замечательно для последовательных задач, кроме того, их тяжело программировать.

# Память

Вторым важным компонентом любого компьютера является память. Идеальная память должна быть чрезвычайно быстрой (быстрее чем исполнение одной операции), достаточно большой и очень дешевой, однако текущие технологии не позволяют добиться всех этих целей, поэтому разные подходы применяются. 

![800](Pasted%20image%2020251105225413.png)

Регистры сделаны из такого же материала, что и процессор, поэтому столь же быстр, чего не скажешь про другие слои памяти.

**Main memory** разделена на кэш-строки, обычно по 64 байта, начиная с 0 до 63 в нулевой строке, с 64 до 127 в первой и тд.  Самые часто используемые кэш-строки располагаются в **кэше**, если строка находится в кэше то устройство берет его оттуда а не от main memory, подобный случай, когда запрос не достигает main memory, а лишь завершается с кэша, называется **cache hit**. Кэш быстрее и дороже main memory.

Существуют несколько уровней кэша. Первый находится внутри процессора и хранит декодированные слова, обычно 32kb, остальные уровни уже снаружи. 

Main memory также еще часто называют **RAM (Random access memory)**, или старое прозвище **core memory**.
